说明：
- 本文件用于多人协作时追踪开发进度。
- 每个任务完成后，把 `[ ]` 改成 `[x]`。
- 每次开始做任务前，先在任务后面写上负责人。
- 每次完成任务后，补一句简短备注，写明改了什么、在哪个分支完成。
- 优先处理 P0，再处理 P1、P2。
- 原则：先跑通，再优化，再扩展。

---

## 一、当前总体目标

当前阶段目标：

**先做出一个可运行、可测试、可协作的最小闭环版本。**

闭环标准：
- 角色能显示
- 背景和地面显示正常
- Player 自动前进
- 点击可跳跃
- 支持二段跳
- 支持长按高跳
- 可触发攻击
- 攻击动画能播放
- 攻击 Hitbox 能命中 Enemy
- Enemy 可死亡
- Camera 能跟随
- 无阻塞性编译错误
- 工程可提交到 GitHub 并被他人正常拉取

## 七、任务流转规则

每个任务按下面流程执行：

1. 认领任务
2. 创建分支
3. 开发与本地测试
4. 更新本文件勾选状态
5. 提交代码
6. 发起 Pull Request
7. 合并到 `dev`
8. 回写备注

备注示例：

- 已完成，分支：`feature/player-attack`
- 已完成，修复 AttackHitbox 重复函数问题
- 已完成，统一 Sorting Layer：Background / Ground / Character / Effect
