# kiyoRun 开发协作规范

本文档用于指导 `kiyoRun` 项目的双人协作开发。  
目标不是复杂流程，而是避免两个人同时修改同一批文件，导致工程混乱、脚本冲突、场景丢失或无法运行。

---

## 一、项目基础信息

项目名称：`kiyoRun`

仓库地址：  
`https://github.com/qtyxqq/kiyoRun`

Unity 版本：  
`6000.4.0f1`

开发工具建议：
- Unity Hub
- Unity Editor 6000.4.0f1
- Visual Studio Code
- Git
- GitHub Desktop（可选）

目标平台：
- iPhone
- iPad

---

## 二、当前阶段的协作原则

当前项目仍处于原型阶段，协作时遵守以下原则：

1. **先保证工程能打开、能运行、能提交。**
2. **先做最小闭环，不同时做过多扩展。**
3. **每次只解决一类问题。**
4. **功能开发优先于美术精修。**
5. **不要直接在 main 分支上开发。**
6. **不要两个人同时改同一个场景文件，除非提前沟通。**
7. **每次提交前必须先自测。**

## 三、新成员环境搭建说明

### 1. 安装基础软件

需要安装以下软件：

- Unity Hub
- Unity 6000.4.0f1
- Visual Studio Code
- Git
- GitHub Desktop（可选）

如果后续需要打 iOS 包：
- macOS
- Xcode
- Unity 的 iOS Build Support 模块

---

### 2. 安装 Unity 版本

在 Unity Hub 中安装：

`Unity 6000.4.0f1`

建议附加模块：
- iOS Build Support
- Documentation（可选）

必须保证协作者使用相同 Unity 大版本和小版本。  
否则很容易出现工程升级、场景差异、包冲突、meta 变更等问题。
